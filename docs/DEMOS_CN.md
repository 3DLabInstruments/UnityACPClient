# 演示场景

宣传视频录制指南。每个演示设计为简短（30-90秒）、视觉效果出色，并展示一项独特功能。

---

## 演示 1："从零搭建场景"（60秒）

**亮点：** 空场景 → 一次对话完成完整环境搭建。

**初始状态：** 新建空 Unity 场景（仅包含 Main Camera + Directional Light）

**提示词序列：**

```
你: 帮我搭一个简单的户外场景，一个绿色的地面，上面放3棵树，2块石头，设置成傍晚的光照

（Agent 将依次调用约8个工具：
  1. create_object → Plane（地面），缩放10倍，绿色
  2-4. create_object → 3个 Cylinder（树干），棕色 + 顶部绿色球体
  5-6. create_object → 2个 Sphere（石头），灰色，随机位置
  7. setup_time_of_day → "sunset" 预设
  8. save）
```

**屏幕展示内容：**
- 左侧：Agent 窗口显示流式响应 + 工具调用折叠面板
- 右侧：Scene 视图实时更新 — 对象逐个出现
- 最终：展示温暖夕阳光照的摄像机角度

**核心卖点：** 零手动操作。自然语言 → 完整场景。

## 演示 2："AI 3D 资产生成"（90秒）

**亮点：** 文字描述 → 3D 模型 → 放置到场景中，全部通过聊天完成。

**初始状态：** 一个简单的房间场景（地板 + 4面墙）

**提示词序列：**

```
你: 用 Meshy AI 生成一把中世纪木椅，然后放到房间中央

（Agent 将：
  1. unity_generate(action="text_to_3d", prompt="medieval wooden chair, game asset")
  2. 轮询任务状态直到完成
  3. instantiate_prefab → 放置生成的模型
  4. set_transform → 位置设为中心 (0, 0, 0)）
```

**接续提示：**

```
你: 现在生成一张配套的木桌，放到椅子旁边
```

**屏幕展示内容：**
- 聊天中显示 Meshy 任务进度
- 生成完成后 3D 模型出现在场景中
- Agent 智能地将对象相对放置

**核心卖点：** AI 生成的 3D 资产，在 Unity 工作流中端到端集成。

## 演示 3："场景查看器 — 询问你的场景"（45秒）

**亮点：** Agent 像一位坐在你旁边的同事一样理解你的场景。

**初始状态：** 一个包含多个对象的现有游戏场景（例如平台跳跃关卡）

**提示词序列：**

```
你: 我的场景里现在有什么？

（Agent 调用：describe → 结构化场景摘要）

你: 哪些对象有 Rigidbody 组件？

（Agent 调用：find_by_component → 列出所有物理对象）

你: Player 的移动设置是什么？

（Agent 调用：get_hierarchy → 找到 Player，
  然后 get_component_data → 读取 PlayerMovement 组件，
  报告 Speed=5, JumpForce=10 等）
```

**屏幕展示内容：**
- Agent 以 Markdown 格式化的结构化数据响应
- 当 Agent 引用对象时，场景中高亮显示对应对象

**核心卖点：** 即时理解项目 — 无需手动点击 Inspector。

## 演示 4："用语言创建粒子系统"（60秒）

**亮点：** 描述视觉效果 → Agent 实时创建并调整粒子系统。

**初始状态：** 暗背景的空场景

**提示词序列：**

```
你: 在场景中央创建一个火焰粒子效果

（Agent 将：
  1. create_object → 空对象 "FireEffect"，附加 ParticleSystem
  2. set_main → duration=5, loop=true, startLifetime=1.5, startSpeed=3, startSize=0.5
  3. set_emission → rateOverTime=50
  4. set_shape → cone, angle=25, radius=0.3
  5. set_color → 从黄色到橙色到透明的渐变
  6. set_size → 随生命周期递减的曲线
  7. preview → play）
```

**接续提示：**

```
你: 让它更猛烈一些 — 更快、更大的粒子，再加点烟雾在上面

（Agent 调整参数 + 创建第二个粒子系统用于烟雾）
```

**屏幕展示内容：**
- Scene 视图中粒子系统实时出现并播放
- 聊天中显示每次参数调整的工具调用
- 前后对比

**核心卖点：** 无需触碰 Inspector 即可创建视觉特效。

## 演示 5："一句话改变光照"（30秒）

**亮点：** 一句话改变整个场景的氛围。

**初始状态：** 一个带家具的房间场景，使用默认光照

**提示词序列（快速连发，5次氛围切换）：**

```
你: 把光照设成早晨
（暖色日出，柔和阴影）

你: 现在切到夜晚
（深蓝环境光，月光方向光）

你: 阴天
（灰色平坦光照，无强烈阴影）

你: 日落
（橙色/粉色天空，长阴影）

你: 回到正午
（明亮白光，极少阴影）
```

**屏幕展示内容：**
- Scene 视图即时在不同氛围间切换
- 聊天中可见每次 `setup_time_of_day` 调用
- 分屏或快速剪切展示全部 5 种氛围

**核心卖点：** 即时美术方向迭代。在 Inspector 中需要几分钟的操作，用 AI 只需几秒。

## 演示 6："修复我的场景 — 批量操作"（45秒）

**亮点：** Agent 一次性诊断并修复多个问题。

**初始状态：** 一个混乱的场景，存在多种问题（对象位置错误、材质缺失、命名不当）

**提示词序列：**

```
你: 看看我的场景 — 有些对象悬在空中，
     你能把它们都放到地面上吗？

（Agent 将：
  1. describe → 分析场景对象
  2. find_by_criteria → 找到 y>0.5 的对象
  3. 多次调用 place_on_ground → 将每个对象射线投射到地形上
  报告："已将 4 个对象放置到地面上"）
```

**接续提示：**

```
你: 现在把所有叫 "GameObject (1)"、"GameObject (2)" 的对象
     根据它们的形状和位置重命名为有意义的名字

（Agent 使用 get_hierarchy + rename 逐个重命名：
  "GameObject (1)" → "LeftWall"
  "GameObject (2)" → "FloorTile_Center"
  等）
```

**屏幕展示内容：**
- Hierarchy 面板实时更新
- 对象吸附到地面
- 层级命名的前后对比

**核心卖点：** 繁琐的清理工作自动化。Agent 做了实习生会做的事，但瞬间完成。

## 演示 7："用语言雕刻地形"（60秒）

**亮点：** 通过描述地理特征来塑造地形，而非手动绘制笔刷。

**初始状态：** 带有一个纹理层的平坦地形

**提示词序列：**

```
你: 在地形中央创建一座山，大约50米高

（Agent 调用：modify_height，使用抬高模式、中心位置、大半径）

你: 在东侧加一个山谷

（Agent 调用：modify_height，使用降低模式、东侧偏移）

你: 在平坦区域涂上草地纹理，山坡上涂岩石纹理

（Agent 调用：paint_texture，低处涂草地，陡坡涂岩石）

你: 在山脚周围种20棵树

（Agent 调用：add_trees，散布位置）
```

**屏幕展示内容：**
- Scene 视图中地形实时变形
- 纹理实时绘制
- 树木逐渐填充

**核心卖点：** 以对话速度进行关卡设计。

## 演示 8："结构化输入 — Elicitation"（45秒）

**亮点：** Agent 通过真实表单精确询问所需信息 — 不再猜测，不再自由文本来回沟通。

**初始状态：** 任意场景。重点在于对话交互，而非场景内容。

**提示词序列：**

```
你: 重构玩家控制器

（Agent 发送 elicitation/create → 内联弹出结构化表单：
  - 下拉框："重构策略" → 保守 / 均衡 / 激进
  - 文本框："补充说明"
  - 预选默认值："均衡"
  用户选择 "激进"，输入 "保持向后兼容"，点击提交）

（Agent 收到类型化数据 {strategy: "aggressive", note: "保持向后兼容"}
  并按照精确参数执行 — 无歧义）
```

**接续展示 Unity 原生字段：**

```
你: 在我的场景中放一个光源

（Agent 发送包含 Unity 特定格式的 elicitation：
  - Vector3Field："位置" → 用户通过原生 Unity 输入选择 (3, 5, -2)
  - ColorField："光照颜色" → 用户通过颜色选择器选取暖橙色
  - PopupField："光源类型" → 点光源 / 聚光灯 / 方向光
  用户填写表单，点击提交 → 按精确参数创建光源）
```

**屏幕展示内容：**
- Elicitation 面板内联显示在聊天消息下方
- 原生 Unity 控件：下拉框、颜色选择器、Vector3 输入框
- 三种操作：提交（接受）、拒绝、取消
- Agent 使用结构化响应直接执行，无需进一步追问

**核心卖点：** 不再猜测，不再解析自由文本。Agent 提出结构化问题，获得类型化答案，精确执行。

## 录制技巧

- **分辨率：** 1920×1080，Unity 使用暗色主题
- **布局：** Agent 窗口停靠在右侧（占 1/3 宽度），Scene 视图在左侧（占 2/3）
- **速度：** 第一次工具调用使用实时速度，后续重复调用使用 2 倍速
- **音频：** 可选旁白或文字字幕说明正在发生的操作
- **字幕：** 同时包含英文和中文提示词以覆盖更广受众
- **时长：** Twitter/X 演示 ≤ 60秒，YouTube 演示 ≤ 90秒，README GIF ≤ 30秒

### 推荐录制顺序

1. **演示 5**（光照）— 最短，视觉冲击力最强，最适合 GIF/推文
2. **演示 1**（搭建场景）— "主打"演示，展示核心价值主张
3. **演示 4**（粒子系统）— 视觉效果出色，本工具独有功能
4. **演示 8**（Elicitation）— 独特的 UX 差异化功能，展示结构化 Agent↔用户交互
5. **演示 3**（场景查看器）— 展示 AI 理解能力，适合开发者
6. **演示 6**（批量修复）— 实用价值，解决日常痛点
7. **演示 7**（地形）— 效果出色但需要地形前期准备
8. **演示 2**（3D 生成）— 依赖 Meshy API，等待时间最长
